Laporan Praktikum 8

Laravel Relationship: Student, Major, dan Subject

Dibuat : 25 Mei 2026

Repository GitHub

Pendahuluan

Dalam membangun aplikasi web yang scalable dengan Laravel, salah satu aspek terpenting adalah kemampuan untuk mengelola hubungan antar data (relationship) dengan baik. Database relasional modern memerlukan pendekatan yang terstruktur untuk menghubungkan berbagai entitas sehingga integritas dan konsistensi data tetap terjaga.

Laravel menyediakan fitur Eloquent ORM (Object-Relational Mapping) yang powerful untuk memetakan hubungan tabel database ke dalam bentuk objek yang mudah digunakan. Melalui pendekatan ini, developer tidak perlu menulis query SQL yang rumit, cukup dengan mendefinisikan method relasi di dalam Model.

Praktikum ini bertujuan memahami dan mengimplementasikan dua jenis relasi fundamental yaitu One-to-Many dan Many-to-Many melalui studi kasus sistem akademik yang menghubungkan data program studi (Major), mahasiswa (Student), dan mata kuliah (Subject).

Tujuan Praktikum

  • Memahami konsep relationship dalam database relasional dan implementasinya di Laravel Eloquent.
  • Mampu membuat file migration dengan proper foreign key constraint dan cascade delete.
  • Mengimplementasikan relasi One-to-Many antara Major dan Student, serta Many-to-Many antara Student dan Subject.
  • Menerapkan teknik Eager Loading untuk mengoptimalkan performa query database.
  • Menggunakan method seperti attach(), sync() untuk mengelola relasi Many-to-Many.
  • Menampilkan data berelasi dengan kompleks di halaman view dengan struktur yang rapi.

Landasan Teori

Relationship atau relasi adalah hubungan antar tabel dalam database yang digunakan untuk menghubungkan data secara logis dan menjaga integritas referensial.

Relasi One-to-Many menggambarkan hubungan di mana satu record dari tabel parent dapat memiliki banyak record di tabel child. Contoh: satu program studi (Major) dapat memiliki banyak mahasiswa (Student). Pada tabel child, disimpan foreign key yang mereferensi primary key tabel parent.

Relasi Many-to-Many menggambarkan hubungan di mana banyak record dari tabel A dapat memiliki hubungan dengan banyak record dari tabel B. Untuk mewujudkan ini, diperlukan tabel perantara (pivot table) yang menyimpan foreign key dari kedua tabel tersebut. Contoh: banyak mahasiswa dapat mengambil banyak mata kuliah.

Eloquent ORM adalah sistem yang mengubah tabel database menjadi objek yang dapat dimanipulasi melalui kode PHP. Dengan method-method seperti hasMany(), belongsTo(), dan belongsToMany(), developer dapat mendefinisikan relasi antar model tanpa query SQL manual.

N+1 problem terjadi ketika melakukan query di dalam loop, menyebabkan banyak database calls yang tidak efisien. Eager Loading menggunakan method with() untuk memuat relasi bersamaan dengan data utama dalam satu query, mengurangi beban database secara signifikan.

Pivot table adalah tabel perantara yang menyimpan dua foreign key. Method attach() digunakan untuk menambahkan relasi baru, sedangkan sync() digunakan untuk sinkronisasi relasi dengan menghapus yang tidak ada dalam array baru dan menambahkan yang baru.

2. Membuat Migration File untuk Tabel Majors

Langkah pertama adalah membangun struktur tabel untuk program studi (Majors). Buat migration file baru: 

php artisan make:migration create_majors_table

Buka file migration yang baru dibuat di database/migrations dan isi dengan skema tabel majors: 

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up()
    {
        Schema::create('majors', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->timestamps();
        });
    }

    public function down()
    {
        Schema::dropIfExists('majors');
    }
};

Tabel majors memiliki kolom id sebagai primary key, name untuk menyimpan nama program studi, serta timestamps untuk mencatat waktu pembuatan dan perubahan data.


3. Membuat Migration File untuk Tabel Students

Selanjutnya buat migration file untuk tabel mahasiswa dengan relasi One-to-Many ke tabel majors: 

php artisan make:migration create_students_table

Isi file migration students dengan kode berikut yang menyertakan foreign key ke tabel majors: 

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up()
    {
        Schema::create('students', function (Blueprint $table) {
            $table->id();
            $table->string('nim')->unique();
            $table->string('name');
            $table->text('address');
            $table->foreignId('major_id')->constrained('majors')->onDelete('cascade');
            $table->timestamps();
        });
    }

    public function down()
    {
        Schema::dropIfExists('students');
    }
};

Pada tabel students, kolom major_id berfungsi sebagai foreign key yang mereferensi tabel majors. Pengaturan onDelete('cascade') memastikan ketika satu program studi dihapus, semua mahasiswa di program studi tersebut akan terhapus otomatis.


4. Membuat Migration File untuk Tabel Subjects

Buat migration file untuk tabel mata kuliah: 

php artisan make:migration create_subjects_table

Isi file migration subjects dengan skema yang menyimpan data mata kuliah: 

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up()
    {
        Schema::create('subjects', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->integer('sks');
            $table->timestamps();
        });
    }

    public function down()
    {
        Schema::dropIfExists('subjects');
    }
};

Tabel subjects menyimpan nama mata kuliah dan jumlah kredit (SKS). Tabel ini akan terhubung dengan tabel students melalui relasi Many-to-Many.


5. Membuat Migration File untuk Tabel Pivot Student_Subject

Untuk mendukung relasi Many-to-Many, buat tabel perantara (pivot table) yang menghubungkan students dan subjects: 

php artisan make:migration create_student_subject_table

Isi file migration dengan struktur pivot table: 

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up()
    {
        Schema::create('student_subject', function (Blueprint $table) {
            $table->id();
            $table->foreignId('student_id')->constrained('students')->onDelete('cascade');
            $table->foreignId('subject_id')->constrained('subjects')->onDelete('cascade');
            $table->timestamps();
            
            // Mencegah kombinasi student_id dan subject_id yang sama tersimpan lebih dari sekali
            $table->unique(['student_id', 'subject_id']);
        });
    }

    public function down()
    {
        Schema::dropIfExists('student_subject');
    }
};

Tabel pivot ini menyimpan dua foreign key yang mereferensi students dan subjects. Constraint unique mencegah seorang mahasiswa terdaftar pada mata kuliah yang sama berkali-kali.

Setelah semua migration file dibuat, jalankan perintah migrate untuk membuat tabel di database: 

php artisan migrate

6. Membuat Model dengan Definisi Relationship

Buat Model Major dengan method relasi ke Student: 

php artisan make:model Major

Buka file app/Models/Major.php dan definisikan relasi One-to-Many: 

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\HasMany;

class Major extends Model
{
    protected $fillable = ['name'];

    public function students(): HasMany
    {
        return $this->hasMany(Student::class);
    }
}

Buat Model Student dengan method relasi ke Major dan Subject: 

php artisan make:model Student

Isi app/Models/Student.php dengan definisi kedua relasi: 

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\BelongsToMany;

class Student extends Model
{
    protected $fillable = ['nim', 'name', 'address', 'major_id'];

    public function major(): BelongsTo
    {
        return $this->belongsTo(Major::class);
    }

    public function subjects(): BelongsToMany
    {
        return $this->belongsToMany(Subject::class);
    }
}

Buat Model Subject dengan method relasi ke Student: 

php artisan make:model Subject

Isi app/Models/Subject.php: 

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsToMany;

class Subject extends Model
{
    protected $fillable = ['name', 'sks'];

    public function students(): BelongsToMany
    {
        return $this->belongsToMany(Student::class);
    }
}

8. Membuat Controller untuk Mengelola Data Relasi

Buat controller untuk menampilkan data mahasiswa dengan relasi: 

php artisan make:controller StudentController

Buka app/Http/Controllers/StudentController.php dan implementasikan method index dengan eager loading: 

<?php

namespace App\Http\Controllers;

use App\Models\Student;
use App\Models\Major;
use App\Models\Subject;
use Illuminate\Http\Request;

class StudentController extends Controller
{
    public function index()
    {
        // Eager loading untuk menghindari N+1 problem
        $students = Student::with(['major', 'subjects'])->get();
        return view('students.index', compact('students'));
    }

    public function create()
    {
        $majors = Major::all();
        $subjects = Subject::all();
        return view('students.create', compact('majors', 'subjects'));
    }

    public function store(Request $request)
    {
        $student = Student::create($request->only(['nim', 'name', 'address', 'major_id']));
        
        // Attach mata kuliah ke mahasiswa
        if ($request->has('subjects')) {
            $student->subjects()->attach($request->subjects);
        }

        return redirect()->route('students.index')->with('success', 'Mahasiswa berhasil ditambahkan');
    }

    public function edit($id)
    {
        $student = Student::with(['subjects'])->findOrFail($id);
        $majors = Major::all();
        $subjects = Subject::all();
        return view('students.edit', compact('student', 'majors', 'subjects'));
    }

    public function update(Request $request, $id)
    {
        $student = Student::findOrFail($id);
        $student->update($request->only(['nim', 'name', 'address', 'major_id']));
        
        // Sync mata kuliah (menghapus yang tidak ada, menambah yang baru)
        if ($request->has('subjects')) {
            $student->subjects()->sync($request->subjects);
        }

        return redirect()->route('students.index')->with('success', 'Mahasiswa berhasil diubah');
    }

    public function destroy($id)
    {
        $student = Student::findOrFail($id);
        $student->delete();

        return redirect()->route('students.index')->with('success', 'Mahasiswa berhasil dihapus');
    }
}

Kesimpulan

Melalui praktikum implementasi Relationship dalam Laravel, telah berhasil dipelajari dan diterapkan konsep-konsep penting dalam mengelola data relasional dengan framework modern.

  • Migration dengan Foreign Key: Pemahaman mendalam tentang cara mendefinisikan hubungan antar tabel melalui file migration dan constraint cascade delete untuk menjaga integritas data.
  • Eloquent ORM: Penggunaan method-method Eloquent seperti hasMany(), belongsTo(), dan belongsToMany() membuat pengelolaan relasi menjadi jauh lebih mudah dibandingkan menulis query SQL manual.
  • Eager Loading: Teknik with() berhasil mengatasi N+1 problem dan meningkatkan performa aplikasi secara signifikan dengan mengurangi jumlah database query.
  • Manajemen Many-to-Many: Method attach() dan sync() memberikan cara yang elegant untuk mengelola relasi kompleks tanpa perlu manipulasi manual tabel pivot.
  • View dengan Blade Template: Blade template memudahkan penampilan data berelasi dengan syntax yang readable dan intuitif melalui akses properti relasional.

Penguasaan konsep relationship di Laravel adalah fondasi penting untuk membangun aplikasi web yang lebih kompleks dengan struktur data yang terorganisir dan efisien. Dengan pemahaman ini, developer siap menghadapi tantangan dalam implementasi sistem informasi skala menengah hingga besar.
